## `parseEdits(input, options?)`

Parse all edits from an LLM response. Returns a sorted array of [`ParsedEdit`](#parsededit)s.
No file I/O.

```ts
import { parseEdits } from "editkit";

const edits = parseEdits(llmOutput);
// [{ format: "search-replace", path: "src/util.ts", search: "...", replace: "...", range: {...} }]
```

Pass `{ formats: ["search-replace"] }` to restrict parsing to one format (useful when
you've prompted the model in one format and want hard-fails on the others).

## `applyEdits(input, files, options?)`

Parse and apply in one async call. Returns one [`ApplyResult`](#applyresult) per parsed
edit, in source order.

```ts
import { applyEdits } from "editkit";

const results = await applyEdits(llmOutput, {
  "src/util.ts": "export const x = 1;\n",
});
```

`files` is either a [`FileMap`](#filemap) or a [`FileReader`](#filereader) function.

## `applyEditsSync(input, files, options?)`

Synchronous variant. Requires a [`FileMap`](#filemap). Useful in tests, in the browser, or
in environments where you've pre-loaded everything into memory.

## `streamEdits(stream, files, options?)` *(in `editkit/ai-sdk`)*

Async iterable that yields `{ edit, result }` for each completed edit as soon as its
closing fence has streamed in.

```ts
import { streamEdits } from "editkit/ai-sdk";

for await (const { edit, result } of streamEdits(textStream, fileReader)) {
  // ...
}
```

See [streaming concepts](/docs/concepts/streaming) for what "complete" means per format.

## `fuzzyReplace(original, search, replace, options?)`

The matching primitive without the parsing layer.

```ts
import { fuzzyReplace } from "editkit";

const out = fuzzyReplace(originalText, searchText, replaceText);
// { kind: "ok", text, strategy: "exact" | "indent-shift" | "trim-eol" }
// { kind: "ambiguous", count: number }
// { kind: "not-found" }
```

## `detectFormats(input)`

Heuristic detector. Returns the formats that appear in `input`, in priority order.

```ts
import { detectFormats } from "editkit";

detectFormats(llmOutput);
// ["search-replace"]  or  ["search-replace", "unified-diff"]  etc.
```

### `ApplyResult`

A discriminated union â€” `{ ok: true, ... }` on success, `{ ok: false, reason, message, ... }`
on failure. See [failure handling](/docs/reference/failure-handling) for the full retry
recipe.

```ts
type ApplyResult =
  | { ok: true; path: string; before: string; after: string; edit: ParsedEdit }
  | { ok: false; path: string; reason: ApplyFailureReason; message: string; edit: ParsedEdit };
```

### `ApplyFailureReason`

```ts
type ApplyFailureReason =
  | "search-not-found"
  | "ambiguous-match"
  | "missing-original"
  | "hunk-context-mismatch"
  | "invalid-format"
  | "io-error";
```

See [failure handling](/docs/reference/failure-handling) for what triggers each code and
how to use them in a retry loop.

### `FileReader`

```ts
type FileReader =
  | FileMap
  | ((path: string) => string | null | Promise<string | null>);
```

Either a sync map or a function. Functions may return `null` (or throw) for paths that
don't exist; the applier treats a missing file as a "create" when `allowCreate` is true and
the format supports creation.
